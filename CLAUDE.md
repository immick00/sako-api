## Architecture

This is a Go REST API (Echo v4) that serves restaurant data — nearby locations via Google Places and menu nutritional info from a Postgres database.

**Request flow:**

1. `GET /nearby?lat=X&lon=Y` → `places.PlacesService.GetRestaurantsAround` → Google Places API (parallel goroutines per restaurant chain) → returns deduplicated, haversine-filtered, distance-sorted results
2. `POST /menus` with `{"restaurants": ["mcdonalds", "subway"]}` → `menus.MenuService.GetMenus` → Postgres via sqlc-generated queries

**Key design decisions:**

- `places/places.go` hardcodes the supported chains (`McDonald's`, `Subway`) and normalizes display names via `restaurantAliases`. The `normalizeRestaurantName` output (e.g. `"mcdonalds"`) is what clients must pass to `/menus`.
- DB code in `db/` is fully generated by sqlc — edit `db/schema.sql` and `db/query.sql`, then run `sqlc generate`. Never edit `db/query.sql.go` or `db/models.go` directly.
- `logger/` wraps slog with a single global `logger.Log` instance used throughout.

## Environment Variables

Required in `.env`:

- `DATABASE_URL` — PostgreSQL connection string (Supabase)
- `GOOGLE_MAPS_API_KEY` — Google Places API key

## Adding a New Restaurant

1. Add table to `db/schema.sql` and query to `db/query.sql`, then run `sqlc generate`
2. Add alias mapping in `places/places.go` `restaurantAliases`
3. Add the chain name to the `restaurants` slice in `GetRestaurantsAround`
4. Add a case to `menus/menus.go` `GetMenus` switch
